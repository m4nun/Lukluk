🐾 Lukluk v1 — Database Schema

12 tables across 6 zones · Row Level Security on all tables · Supabase PostgreSQL

🔐 Auth & Identity
auth.users
Supabase managed — email, password, session tokens
PK id UUID
email, encrypted_password, raw_user_meta_data
→ trigger creates profiles row on signup
profiles
App profile — extends auth.users 1:1
→ auth.users
PK FK id UUID
display_name, avatar_url, timestamps
🗂️ Static Pet Knowledge
pet_type_profiles
Seeded from YAML at deploy time. Global, immutable by users.
PK id TEXT (slug)
name, species, breed, description
📦 mbti_label, mbti_traits
💰 budget_tier, cost ranges
⏱ time_tier, attention hrs
🏠 space_tier, sq_meters
🤧 allergen_level, hypoallergenic
🔊 noise_level, vocal?
⚖️ legality, permits, CITES
🐕 dog/cat/small_pet friendly
✈️ travel, boarding difficulty
🎓 experience_level
🎯 9 matching scores (1–10)
🧠 agent guidance text (5 long fields)
⚠️ concerns JSONB
version, sources
👤 User Data Flow
lifestyle_profiles
Fit Quiz answers → saved once when user gets result
→ profiles
PK id UUID
FK user_id
📋 quiz_answers JSONB
💬 follow_ups JSONB
is_latest, completed_at
match_results
Top 3 pet matches with scores + explanations
→ profiles → lifestyle_profiles
PK id UUID
FK user_id, lifestyle_profile_id
🥇 top_matches JSONB
[rank, scores, explanation]
planning_pet_profiles
Pre-ownership workspace. One per pet type per user.
→ profiles → pet_type_profiles → match_results
PK id UUID
FK user, pet_type, match_result
planning_name, decision_status
💰 estimated_expenses JSONB
✅ concern_checklist JSONB
has_ownership → triggers transition
owned_pet_profile_id (nullable)
owned_pet_profiles
Post-ownership. Converted from planning profile.
→ profiles → planning → pet_type
PK id UUID
FK user, planning, pet_type
pet_name, age_life_stage, got_date
weight_kg, sex, food, health_concerns
💰 actual_expenses JSONB
📅 activity_schedule JSONB
🍖 food_guide JSONB
🤖 Agent System
agent_threads
Maps planning/owned profile → LangGraph thread_id
→ planning_pet_profiles → owned_pet_profiles
PK id UUID
FK planning, owned
🔗 thread_id TEXT UNIQUE
agent_type ("decision"|"care")
is_active
+ LangGraph's auto-created: checkpoints, checkpoint_blobs, checkpoint_writes
agent_drafts
Pending agent edits — user confirms/rejects before save
→ planning_pet_profiles → profiles
PK id UUID
FK planning, user
target (expenses|checklist|status)
📝 current_value JSONB
✏️ proposed_value JSONB
status (pending|confirmed|rejected|stale)
estimated_expenses
Normalized expense rows — agent tool call targets
→ planning_pet_profiles
PK id UUID
FK planning
category, item, amount_thb, note, sort_order
concern_checklist_items
Normalized concern rows — agent tool call targets
→ planning_pet_profiles
PK id UUID
FK planning
concern_id, title, status, note, sort_order
📝 Content & Community
owner_experiences

→ profiles → pet_type_profiles

FK user, pet_type
has_owned, ownership_duration
title, body
is_flagged, flag_reason
Requires active subscription to insert (RLS policy)

subscriptions

→ profiles (UNIQUE)

stripe_customer_id
stripe_subscription_id
status (inactive|active|past_due|canceled)
current_period_start/end

📊 Data Flow


  auth.users ────trigger────▶ profiles
                                     │
                                     ▼
  [Fit Quiz] ────▶ lifestyle_profiles ────▶ match_results
       (quiz_answers + follow_ups)       (top_matches JSONB)
                                               │
                    ┌──────────────────────────┘
                    ▼
  planning_pet_profiles ◄── pet_type_profiles (YAML seed)
       │
       ├──▶ agent_threads ──▶ LangGraph checkpoints (3 tables)
       │         (thread_id)
       │
       ├──▶ agent_drafts
       │         (pending edits → confirm → write to planning or expenses/concerns)
       │
       ├──▶ estimated_expenses
       │
       └──▶ concern_checklist_items
       
                                 [User marks ownership]
                    ┌──────────────────┘
                    ▼
  owned_pet_profiles  ← agent_threads switches to care agent
       │
       └──▶ actual_expenses, activity_schedule, food_guide (all JSONB inline)

  owner_experiences ◄── subscriber-only insert (RLS checks subscriptions)
