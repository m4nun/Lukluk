🎨 YOU NEED TO DESIGN — 4 Global States

Design once, use everywhere. These apply across all pages and components.

DESIGN STATE

🔄 Loading State

Purpose

Shown while data is being fetched from Supabase or the agent API. Must feel fast and reassure the user.

Variants

VariantUse CaseSpec
Page skeletonDashboard, Result, Workspace, Experiences, OwnedMultiple shadcn Skeleton components arranged in the page layout. Cards: rounded rect + circle + 3 bars. Pulsing shimmer animation (animate-pulse).
Table skeletonExpenseTable, ConcernChecklist, Activity Schedule3-5 rows of content bars. Category header bar + item rows with varying widths to look natural.
Inline spinnerButton actions (submit, save, update status)Small spinner icon replacing button text. Button disabled during loading. Use shadcn Spinner or animate-spin on a Loader2 icon.
Chat loadingAgentChat waiting for response3-dot typing indicator ("...") in a message bubble on the agent's side. Dots animate sequentially.

Design Tokens

  • Skeleton bg: bg-muted
  • Animation: animate-pulse (tailwind) — 2s ease-in-out infinite
  • Minimum display time: 300ms (don't flash skeleton if data loads fast — use useDeferredValue or setTimeout debounce)
  • Consistent skeleton shapes: avatar = circle (w-10 h-10), title = w-2/3 h-5, subtitle = w-1/2 h-4, body = w-full h-4

Where Used

  • Dashboard: skeleton profile cards (3-4)
  • Workspace: left panel skeleton + 3 chat bubbles on right

Error Message Guidelines

  • User-facing: plain English, no stack traces. "Could not load workspace. It may have been deleted."
  • Actionable: always offer a next step. "Try again" / "Go back" / "Contact support"
  • Technical fallback: if API returns unknown error, show generic "Something went wrong. Please try again."
  • Network errors: "Could not connect. Check your internet connection."

Auth Flow

  1. User clicks "Sign in with Google" → redirect to Google OAuth
  2. Google authenticates → callback to /auth/google/callback
  3. Supabase creates/updates user record
  4. Session cookie set → user redirected to /quiz (first time) or /dashboard (returning)
