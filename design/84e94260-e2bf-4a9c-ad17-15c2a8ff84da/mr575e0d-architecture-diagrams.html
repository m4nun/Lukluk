1. System Overview

Lukluk is a Next.js monolith deployed on Vercel with Supabase (PostgreSQL + Auth) and OpenRouter (LLM). The Decision Agent and Care Agent run inside Next.js API routes via LangGraph StateGraphs.

Database: Supabase
LLM: OpenRouter (DeepSeek)
Hosting: Vercel
Auth: Google OAuth only
Payments: Stripe test mode

6. Quiz Flow

The quiz has 9 fixed questions followed by optional LLM follow-up rounds (capped at 20 total). Answers are tracked in sessionStorage during the quiz and saved to Supabase only when the user gets their result.

sequenceDiagram actor User participant Quiz as /quiz (Client) participant MatchAPI as POST /api/match participant FollowUp as POST /api/match/follow-up participant Engine as Matching Engine (Pure) participant LLM as OpenRouter (DeepSeek) participant Supabase as Supabase User->>Quiz: Start Quiz loop 9 Questions Quiz->>User: Show question (budget, time, space, allergy, noise, travel, pets, experience, children) User->>Quiz: Select answer end Quiz->>MatchAPI: POST { answers } MatchAPI->>Engine: runMatch(profiles[], lifestyle) Engine->>Engine: Score all 19 profiles × 8 dimensions Engine-->>MatchAPI: Top 3 matches MatchAPI->>Supabase: Save lifestyle_profiles + match_results MatchAPI-->>Quiz: { matchResultId, matches } Quiz->>User: Show results → /result/[id] opt LLM Follow-ups (if confidence < 80 or close scores) User->>FollowUp: POST { answers, followUps } FollowUp->>Engine: runMatch(profiles[], lifestyle) FollowUp->>LLM: "Generate up to 3 clarifying questions" LLM-->>FollowUp: Question list FollowUp-->>User: Follow-up questions loop Until confidence or 20 cap User->>FollowUp: Answer follow-ups FollowUp->>Engine: Re-run match FollowUp->>LLM: Generate more questions if needed end end

10. Agent Draft Workflow

Agent tool calls never write directly to the database. They create drafts. The user confirms or rejects. Manual edits discard stale drafts.

sequenceDiagram actor User participant Chat as AgentChat participant API as POST /api/agent/chat participant Graph as LangGraph Agent participant DraftRepo as DraftPlanningRepository participant DraftStore as SupabaseDraftStore participant RealRepo as SupabasePlanningRepository participant DraftPanel as DraftPanel participant DB as Supabase User->>Chat: "Update my expenses" Chat->>API: POST { planningProfileId, message } API->>Graph: agent.invoke({ messages: [HumanMessage] }) Graph->>Graph: LLM decides: call update_expenses tool Graph->>DraftRepo: replaceExpenses(pid, newExpenses) DraftRepo->>RealRepo: getExpenses(pid) — snapshot current RealRepo-->>DraftRepo: current expenses DraftRepo->>DraftStore: create({ target, current_value, proposed_value }) DraftStore->>DB: INSERT into agent_drafts (status: pending) Graph-->>API: "I've proposed expense updates" API-->>Chat: Response text Note over DraftPanel,DB: DraftPanel polls every 5 seconds DraftPanel->>DB: SELECT pending drafts alt User accepts User->>DraftPanel: Click "Accept" DraftPanel->>API: POST /api/agent/drafts { draftId, action: "confirm" } API->>DB: UPDATE draft status = confirmed API->>RealRepo: replaceExpenses(pid, proposed_value) RealRepo->>DB: WRITE to estimated_expenses else User rejects User->>DraftPanel: Click "Reject" DraftPanel->>API: POST { draftId, action: "reject" } API->>DB: UPDATE draft status = rejected else User manually edits User->>Chat: Manual edit via left panel Chat->>API: Discard stale drafts for that target API->>DB: UPDATE drafts status = stale end
